# 《数据素养课程》H5网站

## 网站功能

- 介绍课程目的和内容
- 介绍讲师
- 教学案例
- 联系方式和体验链接

## 内容排版

响应式设计，每个内容板块占满一屏。

### 页头和页脚

1. **页头**：
    - 网站Logo
    - 汉堡包菜单：首页、课程、资源、关于我们
2. **页脚**：
    - 社交媒体链接：微博、公众号、领英、GitHub
    - 联系方式：地址、电话、邮箱
    - 版权及网站备案信息
3. **页头/页脚渲染方式**：
    - 页头：以浮层方式展现，固定在屏幕顶端
    - 页脚：作为最后一屏内容进行展示

## 设计风格

### 整体风格
- **现代简洁**：采用扁平化设计风格，避免过多装饰元素，强调内容的可读性和信息层次。
- **数据驱动**：融入数据可视化元素，如图表、图标和统计图形，突出课程的科技感和专业性。
- **移动优先**：响应式设计，确保在手机、平板和桌面设备上均有良好体验。

### 颜色方案
- **主色调**：深蓝色 (#1E3A8A) - 代表科技、信任和专业性。
- **辅助色**：浅蓝色 (#3B82F6) 用于按钮和链接，灰色 (#6B7280) 用于次要文字，白色 (#FFFFFF) 作为背景。
- **强调色**：橙色 (#F59E0B) 用于重要按钮和数据高亮。

### 字体
- **标题字体**：使用无衬线字体，如 "PingFang SC" 或 "Helvetica Neue"，字号 24-48px，加粗。
- **正文字体**：使用易读的无衬线字体，如 "PingFang SC" 或 "Arial"，字号 14-18px，行高 1.6。
- **特殊字体**：数据统计数字使用等宽字体，如 "Monaco" 或 "Consolas"，突出数字感。

### 布局与结构
- **分屏滚动**：采用全屏滚动布局，每屏聚焦一个主题，避免页面过长。
- **网格系统**：基于 12 列网格，确保内容对齐和视觉平衡。
- **间距**：使用一致的间距单位（如 8px 倍数），保证视觉层次清晰。

### 视觉元素
- **图标**：使用简洁的线条图标，风格统一，如 Feather Icons 或 Material Icons。
- **图片**：高清照片，风格现代，突出人物和场景的专业感。
- **数据可视化**：使用简洁的图表，如柱状图、饼图，颜色与整体配色一致。
- **动画效果**：轻微的淡入淡出和滚动动画，提升用户体验，但避免过度。

### 交互设计
- **按钮样式**：圆角矩形，主色调填充，悬停时有阴影效果。
- **导航**：汉堡包菜单展开时平滑动画，页脚链接有悬停高亮。
- **响应式**：触摸友好的按钮尺寸，最小 44px 高度。

### 品牌一致性
- **Logo**：简洁的文字或图形 Logo，融入网站顶部。
- **一致性**：所有元素遵循上述风格，确保网站整体协调。

## 代码风格和项目结构

### 项目结构
- **根目录**：index.html（主页面）、README.md（项目文档）
- **css/**：存放样式文件，如 main.css（主样式）、components.css（组件样式）
- **js/**：存放JavaScript文件，如 app.js（主脚本）、utils.js（工具函数）
- **images/**：存放图片资源，如 logo.png、banner.jpg
- **assets/**：其他资源，如 fonts/（字体）、icons/（图标SVG）

### 代码风格
- **HTML**：
  - 使用语义化标签（如 `<header>`, `<main>`, `<section>`）
  - 缩进使用4个空格
  - 属性值使用双引号
  - 避免内联样式和脚本
- **CSS**：
  - 使用Scss编写
  - 使用BEM命名法（如 .block__element--modifier）
  - 属性按字母顺序排列
  - 使用rem/em单位，基准font-size为16px
  - 避免使用!important，除非必要
- **JavaScript**：
  - 使用ES6+语法（const/let、箭头函数、模板字符串）
  - 变量和函数使用camelCase命名
  - 构造函数使用PascalCase
  - 添加JSDoc注释说明函数用途和参数
- **通用规范**：
  - 文件名使用kebab-case（如 main-style.css）
  - 代码行长度不超过120字符
  - 提交前运行代码检查工具（如ESLint、Prettier）

## 工具链

- 包管理器：npm
- 构建工具：vite
- 代码检查工具：ESLint
- 代码格式化工具：Prettier
- 部署工具：GitHub Pages，直接从GitHub仓库部署静态网站

## 开发环境设置

1. **安装Node.js**：下载并安装最新LTS版本（https://nodejs.org/）。
2. **克隆仓库**：`git clone https://github.com/zeng-hongwei/huangchen.github.io.git`
3. **安装依赖**：进入项目目录，运行 `npm install`。
4. **启动开发服务器**：运行 `npm run dev`（使用Vite开发服务器，支持热重载）。
5. **代码检查**：运行 `npm run lint` 检查代码质量，`npm run format` 格式化代码。

## 构建和部署

1. **构建生产版本**：运行 `npm run build`，生成 `dist/` 文件夹。
2. **本地预览**：运行 `npm run preview` 检查构建结果。
3. **部署到GitHub Pages**：推送代码到main分支，GitHub Actions自动部署（需配置workflow文件）。

## 浏览器兼容性

- 支持现代浏览器：Chrome 90+、Firefox 88+、Safari 14+、Edge 90+。
- 移动端：iOS Safari、Android Chrome。
- 不支持IE浏览器。

## 性能优化

- **图片优化**：使用WebP格式，压缩图片大小。
- **代码分割**：利用Vite的代码分割功能，按需加载JS。
- **懒加载**：对图片和非首屏内容使用懒加载。
- **缓存策略**：设置适当的HTTP缓存头。
- **目标**：首屏加载时间 < 3秒，整体页面大小 < 2MB。

## 测试策略

- **手动测试**：在不同设备和浏览器上检查功能和布局。
- **自动化测试**：使用Jest进行JavaScript单元测试（如果有复杂逻辑）。
- **性能测试**：使用Lighthouse评估性能、可访问性和SEO。
- **跨设备测试**：使用BrowserStack或类似工具。
